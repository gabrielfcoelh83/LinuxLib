| > >> &&
Operadores de encadeamento e redirecionamento.
Símbolos
ls | grep ".txt"

| pipe: saída do ls vira entrada do grep.

&&: cada comando só executa se o anterior foi bem-sucedido.

Redireciona stderr+stdout, salva em arquivo e filtra resultado — tudo em uma linha.

  • | Pipe: saída → entrada
  • > Redireciona (sobrescreve)
  • >> Redireciona (append)
  • && Executa se anterior OK
  • || Executa se anterior falhou
  • 2>&1 stderr → stdout
  • ; Sequencial (ignora erro)

  • * Zero ou mais chars
  • ? Um char qualquer
  • [abc] Um dos listados
  • [a-z] Intervalo
  • [!abc] Nenhum dos listados

7 cards
grep
Buscar padrões de texto com expressões regulares.
Busca

Busca a palavra e exibe as linhas encontradas.

-r recursivo, -n número de linha, -i case-insensitive.

Regex estendida -E: encontra entradas de log com data ISO.

  • -v Inverte: linhas que NÃO batem

find
Localizar arquivos com critérios avançados.
Busca
find . -name "*.pdf"

Localiza todos os PDFs a partir do diretório atual.

find /var/log -name "*.log" -mtime -7 -size +1M

Logs maiores que 1MB modificados nos últimos 7 dias.

find . -type f -name "*.js" -exec grep -l "console.log" {} \;

Arquivos JS que contêm console.log — útil antes de abrir PR.

⚠️-delete é irreversível. Teste sempre com -ls antes.
  • -type f → apenas arquivos
  • -type d → apenas diretórios
  • -maxdepth 2 → limitar profundidade

chmod
Alterar permissões de acesso a arquivos.
Permissões

Adiciona permissão de execução para todos.

chmod -R u=rwX,g=rX,o= /var/www/html

Simbólico recursivo: dono lê/escreve, grupo lê, outros sem acesso. X aplica exec só em diretórios.

CharDecPermissão

--x1Execução
